This guide will take you from zero to running your first HTTP request with Mailman.

<Steps>
  <Step title="Install Mailman">
    Install Mailman globally using your package manager:

    <CodeGroup>
    ```bash npm
    npm install -g mailman-cli
    ```

    ```bash yarn
    yarn global add mailman-cli
    ```

    ```bash pnpm
    pnpm add -g mailman-cli
    ```
    </CodeGroup>
  </Step>

  <Step title="Initialize a Project">
    Create a new directory and initialize Mailman:

    ```bash
    mkdir my-api-project && cd my-api-project
    mailman init
    ```

    You'll see output like:

    ```
    Created config.json
    Created requests/ directory
    Created requests/get-users.json
    Created requests/create-user.json

    Mailman initialized successfully!
    Run 'mailman list' to see available requests
    ```

    <Tip>
      The `init` command creates sample request files and a configuration that you can customize.
    </Tip>
  </Step>

  <Step title="Explore Sample Requests">
    List all available requests:

    ```bash
    mailman list
    ```

    You'll see:

    ```
    Available requests:

      Get Users
      requests/get-users.json

      Create User
      requests/create-user.json
    ```

    Let's examine the GET request that was created:

    ```json requests/get-users.json
    {
      "name": "Get Users",
      "method": "GET",
      "url": "{{baseUrl}}/users",
      "headers": {
        "Authorization": "Bearer {{API_KEY}}"
      }
    }
    ```

    <Note>
      Notice the `{{baseUrl}}` and `{{API_KEY}}` variables. These are resolved from your environment configuration.
    </Note>
  </Step>

  <Step title="Preview a Request">
    Before running a request, you can preview how variables will be resolved:

    ```bash
    mailman view requests/get-users.json
    ```

    Output:

    ```
    Request: Get Users
    Environment: dev

    Method: GET
    URL: http://localhost:3000/users

    Headers:
      Authorization: Bearer {{API_KEY}}
    ```

    <Tip>
      Use the `-e` flag to preview with a different environment: `mailman view requests/get-users.json -e prod`
    </Tip>
  </Step>

  <Step title="Run Your First Request">
    Execute the request interactively:

    ```bash
    mailman run
    ```

    This will:
    1. Show you a list of available requests
    2. Let you select one with arrow keys
    3. Prompt you for any missing variables (like `{{API_KEY}}`)
    4. Execute the request and display the response

    Or run a specific request directly:

    ```bash
    mailman run requests/get-users.json
    ```

    <Warning>
      If your server isn't running at `http://localhost:3000`, you'll see a "Connection refused" error. Update the `baseUrl` in `config.json` or start your local server.
    </Warning>
  </Step>

  <Step title="Customize Your Environment">
    Edit `config.json` to match your API:

    ```json config.json
    {
      "defaultEnvironment": "dev",
      "environments": {
        "dev": {
          "variables": {
            "baseUrl": "http://localhost:3000"
          }
        },
        "prod": {
          "variables": {
            "baseUrl": "https://api.example.com"
          }
        }
      }
    }
    ```

    Switch environments using the `-e` flag:

    ```bash
    mailman run requests/get-users.json -e prod
    ```
  </Step>
</Steps>

## Making a POST Request

Let's examine the POST request that was created:

```json requests/create-user.json
{
  "name": "Create User",
  "method": "POST",
  "url": "{{baseUrl}}/users",
  "headers": {
    "Authorization": "Bearer {{API_KEY}}",
    "Content-Type": "application/json"
  },
  "body": {
    "name": "John Doe",
    "email": "john@example.com"
  }
}
```

Run it:

```bash
mailman run requests/create-user.json
```

## Using Environment Variables

Create a `.env` file for sensitive data:

```bash .env
API_KEY=your-secret-api-key
DEBUG=true
```

Create environment-specific files:

```bash .env.prod
API_KEY=prod-secret-key
```

<Note>
  Variables are resolved in this priority order:
  1. Runtime `-v` flags
  2. `.env.<environment>` file
  3. `.env` file
  4. Request `defaultValues`
  5. Environment variables from `config.json`
</Note>

## Overriding Variables at Runtime

Override any variable when running a request:

```bash
mailman run requests/get-users.json -v baseUrl=http://localhost:8080 -v API_KEY=test-key
```

You can use `-v` multiple times to set multiple variables.

## Creating New Requests

Create a new request interactively:

```bash
mailman new
```

This will:
1. Prompt you to select a directory (or create a new one)
2. Guide you through request configuration
3. Save the request as a JSON file

## Response Output

When you run a request, you'll see:

```
Running: Get Users
Environment: dev
GET http://localhost:3000/users

Status: 200 OK
Time: 45ms

Response Headers:
  content-type: application/json
  content-length: 1234

Response Body:
[
  {
    "id": 1,
    "name": "John Doe",
    "email": "john@example.com"
  }
]
```

<Tip>
  Status codes are color-coded:
  - **Green**: 2xx Success
  - **Yellow**: 3xx Redirects
  - **Red**: 4xx and 5xx Errors
</Tip>
